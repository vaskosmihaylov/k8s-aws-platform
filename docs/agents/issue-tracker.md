# Issue tracker

Work is tracked in this repository's GitHub Issues using `gh`.

- Read-only discovery: `gh issue list`, `gh issue view`, `gh pr list`, `gh pr view`, `gh pr diff`.
- Creating or modifying issues, comments, labels, pull requests, or relationships is externally visible and requires explicit user approval.
- `to-spec` and `to-tickets` show drafts before publishing.
- Tickets use GitHub's native blocking/sub-issue relationships when available; otherwise the issue body records `Blocked by` links.
- Pull requests are not automatically treated as incoming triage work.
